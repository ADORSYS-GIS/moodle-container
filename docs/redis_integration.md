# Redis Integration in Moodle

## Overview
Redis is used by Moodle as a high‑performance, in‑memory cache and session store. It reduces database load, speeds up page rendering, and provides a robust locking mechanism for background tasks.

## What the Docker stack provides
- A **Redis container** (`redis:7-alpine`) exposing port `6379` only on `127.0.0.1` for security.
- Environment variables are passed to the Moodle container (`REDIS_*`) so the **Moodle init script** can inject the correct values into `config.php`.
- The following stores are pre‑configured in the default `config.php` by the init script (see `base/opt/setup_moodle.sh`):
  - **Session store** – `session_redis` (stores PHP session data)
  - **Application cache** – `redis` (general cache for the cache API)
  - **Lock store** – `redislock` (used by Moodle cron and background tasks)

## How to enable and configure Redis in Moodle
1. **Set the environment variables** (already defined in `compose.yaml`):
   ```yaml
   REDIS_SESSION_ID_HOST: redis
   REDIS_SESSION_ID_PORT: 6379
   REDIS_SESSION_ID_AUTH_STRING: "moodledevpass"
   REDIS_SESSION_IP_AND_PORT: redis:6379
   REDIS_SESSION_AUTH_STRING: "moodledevpass"
   REDIS_APP_IP_AND_PORT: redis:6379
   REDIS_APP_AUTH_STRING: "moodledevpass"
   REDIS_LOCK_HOST_AND_PORT: redis:6379
   REDIS_LOCK_AUTH_STRING: "moodledevpass"
   ```
   These values are automatically written to `config.php` by `setup_moodle.sh`.

2. **Log into Moodle as an admin** and navigate to **Site administration → Plugins → Cache → Configuration**.
   - **Session cache**: Choose **Redis** and set the *Server URL* to `redis:6379`. Use the *Password* field if you defined one (`moodledevpass`).
   - **Application cache**: Select **Redis** for the *Default cache store*.
   - **Lock store**: Choose **Redis** for the *Lock store*.

3. **Verify the configuration**
   - Go to **Site administration → Server → Environment** – you should see a *Redis* entry marked as *OK*.
   - Check the *Cache store status* page – all three stores (session, application, lock) should show **Connected**.

## Advanced tweaks (optional)
| Setting | Description | Example |
|---------|-------------|---------|
| `session_redis_prefix` | Prefix for session keys to avoid collisions when sharing a Redis instance with other apps. | `mdl_sess_` |
| `redis_max_concurrency` | Maximum concurrent connections from Moodle. Adjust if you see *Too many connections* errors. | `30` |
| `redis_persistent` | Use persistent connections (recommended for Docker). | `yes` |

> **Tip**: If you need to change any of these values after the container is running, edit `config.php` directly or update the corresponding environment variables and recreate the containers.
